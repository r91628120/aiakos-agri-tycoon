# AIAKOS FarmVerse Development Rules

本專案正式名稱：

- 中文名稱：AIAKOS 農場宇宙
- 英文名稱：AIAKOS FarmVerse

## 開發基準

- 修改前必須先閱讀 `docs/hand_off.md`，並以該文件作為最高開發依據。
- 一次只處理一個明確任務；不要在同一輪混入額外功能、重構或設計改版。
- 不得任意刪除現有功能、流程、資料或 UI 狀態。
- 不得未經使用者同意更換框架、改變技術架構或安裝新套件。
- 不得修改與當前任務無關的檔案。
- 修改前後都要執行可用的檢查，例如安裝、啟動、build、lint、test；若環境限制導致無法執行，必須明確回報原因。
- 所有功能必須同時考量桌機與手機版體驗。
- 介面文字與回報內容必須維持繁體中文。
- 編輯檔案時必須確認文字編碼正常，避免繁體中文內容出現亂碼。
- 每次完成後必須回報修改檔案、測試結果及未完成事項。

## 限制事項

- 不要重新設計 UI，除非使用者明確要求。
- 不要一次新增多種作物；第一版以番茄完整流程為優先。
- 不要新增資料庫或外部服務，除非使用者明確批准。
- 不要發布、部署、推送到 GitHub 或建立 PR，除非使用者明確批准。
