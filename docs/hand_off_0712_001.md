## 3. 已完成的功能

- 暱稱、農場名稱、課堂代碼輸入。
- 番茄作物可玩，其他作物只顯示「即將開放」。
- 可選種植面積：0.5、1、2、3 公頃。
- 生長按鈕每次有效點擊推進 1 點，並顯示 `+1` 飄字。
- 生長進度 0 到 100，進度不會自動增加。
- 20、40、60、80、100 事件節點已存在。
- 事件未完成時照顧按鈕會鎖定。
- 20、40、80 事件使用決策 Modal；60 使用災害輪盤；100 進入採收人力。
- 事件與輪盤結果會寫入 `handled`，避免同一節點重複領取。
- 每 30 次有效點擊顯示鼓勵訊息，且以 `lastCheerIndex` 避免連續同句。
- 採收工人 1 到 5 人，採收成本與完成率會影響實際採收量。
- 品質分級會產生 A 級、B 級、加工品、淘汰品，比例加總為 100%。
- 可選冷鏈：不使用冷鏈、租用冷鏈、自建冷鏈。
- 可選銷售通路：批發市場、網路品牌直售、日本精品外銷。
- 結算會計入冷鏈成本、物流成本、冷鏈損耗、通路倍率與營收。
- `localStorage` 自動保存目前 gameState。
- 手機版已有單欄配置、底部 dock、Modal 高度與按鈕遮擋修正。

## 7. 目前遊戲流程

目前實作流程如下：

1. `login`：輸入暱稱、農場名稱、課堂代碼。
2. `setup`：番茄為可玩作物，其他作物鎖定；選擇種植面積。
3. `grow`：照顧農場，生長值由 0 推進到 100。
4. 生長到 20：幼苗定植事件，玩家選四個決策之一。
5. 生長到 40：整枝與支架事件，玩家選四個決策之一。
6. 生長到 60：病蟲害輪盤，隨機給予損失或利多。
7. 生長到 80：極端天氣事件，玩家選四個決策之一。
8. 生長到 100：採收時刻，進入採收人力。
9. `workers`：選採收工人 1 到 5 人，確認後計算採收量。
10. `quality`：轉動品質輪盤，計算 A、B、加工、淘汰重量。
11. `cold`：選冷鏈方案。
12. `sales`：選銷售通路。
13. `result`：顯示本季淨利、實際採收量與 A 級品重量。

## 9. 已知問題與技術風險

- `app/page.tsx` 已從極短單檔原型擴成大型單檔，後續維護風險高，但目前依限制未拆檔。
- `app/layout.tsx` metadata 仍為 `Starter Project`，語系仍是 `lang="en"`，尚未符合正式產品名稱與繁體中文語系。
- 目前品質分級是簡化輪盤，尚未依 `docs/hand_off.md` 完整做 A/B/加工多段輪盤權重。
- 目前銷售沒有真正輪盤；選通路後直接結算。
- 目前「日本精品外銷」只是銷售通路名稱，尚未包含外銷國家、檢疫、運輸方式。
- `npm run build`、`npm run lint`、`npm test` 在目前 Windows shell 失敗，因 scripts 依賴 `bash`。
- `tests/rendered-html.test.mjs` 期待 `worker.fetch`，但目前 `dist/server/index.js` 的 default export 是 function，`default.fetch` 為 `undefined`。
- `tsc --noEmit` 會產生 `tsconfig.tsbuildinfo`，需在交付前清理或加入 ignore。
- Git 提醒 `app/page.tsx`、`app/globals.css` 在 Windows 下可能 LF 轉 CRLF，後續需注意編碼與換行。
- 尚未用瀏覽器實機驗證手機畫面，只完成 CSS 層級修正與靜態/建置檢查。

## 10. build、lint、test 執行結果

在目前 Windows / PowerShell 環境執行既有 npm scripts：

- `npm.cmd run build`：失敗。
  - 原因：`bash scripts/build-verified.sh`
  - 錯誤：`'bash' is not recognized as an internal or external command`
- `npm.cmd run lint`：失敗。
  - 原因：`bash scripts/sites-env.sh -- eslint ...`
  - 錯誤：`'bash' is not recognized as an internal or external command`
- `npm.cmd test`：失敗。
  - 原因：先呼叫 `npm run build`，同樣卡在缺少 `bash`。

直接執行本地工具的結果：

- `npx.cmd tsc --noEmit`：通過。
- `.\\node_modules\\.bin\\eslint.cmd . --ignore-pattern dist --ignore-pattern .next`：通過。
- `$env:WRANGLER_LOG_PATH='.wrangler/wrangler.log'; .\\node_modules\\.bin\\vinext.cmd build`：通過。
- `node --test tests/rendered-html.test.mjs`：失敗。
  - 錯誤：`TypeError: worker.fetch is not a function`
  - 觀察：`dist/server/index.js` 匯出 keys 為 `default`、`generateStaticParamsMap`；`typeof default` 是 `function`；`typeof default.fetch` 是 `undefined`。
